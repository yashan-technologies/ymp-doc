## v23.5.7.2

### Release Date

March 4, 2026.

### New Features

2. Supports startup validation interception for the deployment configuration file application.properties.
4. Supports evaluation and adaptation for virtual columns in YashanDB version 23.4.7.
6. Supports pre-check of USE_NATIVE_TYPE configuration for the evaluation library and target library.
8. Optimizes permission configuration and documentation for each stage of the task.
10. Optimizes the display of the number of rows checked in the validation process.
12. Optimizes the logging of the validation process.

### Bug Fixes

2. Fixes the issue that the ymp_memory parameter in the comment does not take effect at startup.
4. Fixes the issue that the thread for checking the completion of sharding validation does not close, leading to slow validation.
6. Fixes the issue that the pre-check for independent migration download constraint files on sharded tables throws an error.
8. Fixes the issue that the evaluation process throws an error when the tablespace length exceeds the range.
10. Fixes the issue that the Y2Y evaluation process throws an error when converting constraint content.

### Release Date

December 03, 2025.

### New Features

1. Support for user DDL with default tablespace attributes during assessment of all sources.

4. Fixed the issue with displaying the progress of validation tasks.
5. Fixed the confusion in the display of hidden column lists in the object list.
6. Fixed the inconsistency in validation of M2YM time types.
7. Fixed the inconsistency in the validation of sorting keys with special characters across various sources.
