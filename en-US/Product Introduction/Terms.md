|Term |Description |
| ------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Data source     | Data source mainly includes the database connection information that needs to be evaluated and migrated, which is added by the user and provided for the task.                                                                                                                                                              |
| Assessment before migration    | A phase of the task that evaluates the metadata compatibility of the source database with YashanDB, and adapts incompatible parts to prepare for migration.                                                                                                                                                     |
| Data Migration | A phase of the task, involving the migration of metadata and data from the source database to the target database YashanDB. There are 3 main steps:<br/><ul><li>Object migration: Migration scope: all definition-level objects such as tables, views, stored procedures, functions, and package bodies.</li><li>Full data migration: Before the task starts, migrate all existing data from the source end up to the current moment to the target end in one go to ensure the initial data consistency.</li><li>Supplementary object integrity: Supplementary scope: integrity objects that depend on data or are interdependent, such as primary key constraints, unique constraints, foreign key constraints, indexes, and triggers.</li></ul> |
| Consistency check   | A phase of the task that formats the data of the source database, formats the data of the target database in the same way, compares the two results, and displays the details of the comparison results.                                                                                                                                                                                                |
| Source database   | The source database for metadata and data that is to be evaluated and migrated in the task.                                                                                                                                                                                                                                              |
| Target database   | The target database for data migration in the task.                                                                                                                                                                                                                                                        |
| Built-in database   | The built-in database is an internal background database of the YMP service, used to host all business data necessary for the normal operation of the YMP service, including user management, data source management, task status, and other information. It can be divided into default built-in database and custom built-in database: <br/><ul><li>default built-in database: A built-in library specified to be installed together with YashanDB during the installation of YMP, referred to as the default built-in database.</li><li>custom built-in database: A pre-existing YashanDB database specified as a built-in library during YMP installation, referred to as the custom built-in database.</li></ul>                                            |
| Assessment database   | The assessment database is used to evaluate the compatibility of migrated metadata during the assessment before migration phase. It is divided into assessment built-in database and assessment custom database: <br/><ul><li>assessment built-in database: Indicates the use of the existing built-in database (the built-in database mentioned in the previous term) as the assessment database, thus called assessment built-in database.</li><li>assessment custom database: Indicates an externally available YashanDB specified by the user as the assessment database (if an external database is specified, it must have the same version as the target database and cannot be used as the target database), thus called assessment custom database.</li></ul>                                   |
| yasldr | Yasldr is the built-in data import tool of YashanDB, which YMP relies on for data migration. This tool is installed as a component of YMP during the installation of YMP.                                                                                                                                                                                                        |
| schema | This document stipulates that "database" in MySQL is referred to as "schema."                                                                                                                                                                                                                                     |
