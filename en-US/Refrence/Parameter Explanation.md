In the project extraction directory, before starting YMP, you can configure certain parameters according to your needs. By modifying these parameters, you can adapt to specific user requirements and ensure the normal execution of YMP tasks.

> **Note**:
>
> After modifying any of the following parameters, YMP must be restarted for the changes to take effect. For detailed restart procedures, please refer to [Restart YMP Command Reference](../Installation and Upgrade/Installing and Uninstalling/Installing and Uninstalling(default).html#StopYMP).

### properties-validation-rules.yml

When YMP starts, it validates the parameters in the `conf/application.properties` to proactively prevent runtime exceptions caused by incorrect configurations.

- Externalization of Validation Rules：YashanDB centrally maintains all parameter validation rules in an independent configuration file, `properties-validation-rules.yml`, supporting external management and custom overriding of rules. The specific mechanism is as follows:
  - Built-in Rules (Default): YMP includes a complete set of validation rules, which can be used directly without additional configuration.
  - External Rules: Users can create a new file with the same name, `properties-validation-rules.yml`, in the `conf/`directory according to actual needs, and define or modify validation rules within it following the prescribed format. After the file is created:
    - YMP will automatically load the rules defined in the user's file.
    - User-defined rules can override the built-in rules for corresponding parameters.
    - Parameters not defined in the user's file will continue to follow the system's built-in rules.
- Diversification of validation rule parameters:

  - Non-null validation:
    - Required parameters: `server.port`, `time_zone_id`, `spring.datasource.url`, `spring.datasource.username`, `spring.datasource.password`, etc.
    
    - Optional parameters: `ymp.logging.level`, `spring.datasource.defaultTableType`, `migration.character_set`, `migration.national_character_set`, etc.

  - Numeric range validation: Validate valid ranges for numeric parameters and report an error if out of range.

  - Format validation: Support validation for types such as time zone ID, CRON expressions, enumeration values, and boolean values.
