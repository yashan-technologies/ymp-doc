The operation and maintenance monitoring of DMC includes three parts: operation log monitoring, alarm log monitoring, and various metric monitoring.

At the same time, DMC supports integration with Prometheus monitoring software to enable real-time monitoring of system operational status, collecting performance metrics, and conducting data analysis and alerts through the rich functionality provided by Prometheus.

## Operation Log Configuration

DMC provides the following three types of operation logs to record various event information generated during the operation of DMC:

**Server Operation Log**

The server operation log is the `run.log` file located in `{DMC installation directory}/logs/{DMC instance name}`, used to record a series of events that occur during the interaction between the server and client.

This log is generated by default and cannot be configured or modified.

**Client Operation Log**

The client operation log is the `client.log` file located in `{DMC installation directory}/logs`, used to record a series of events generated by commands executed on the client.

This log is generated by default and cannot be configured or modified.

**Task Operation Log**

The task operation log is the `run.log` file located in `{DMC installation directory}/logs/{DMC instance name}/{migration task name}{timestamp}`, where `{timestamp}` is the SCN formatted as `yyyyMMddHHmmss`, used to record a series of events generated during the data synchronization process of DMC migration tasks.

Users can specify the following levels for task operation logs in the server startup configuration file (`{DMC installation directory}/conf/yds_config.yml`) according to their actual needs:

- Trace level is the most detailed log level, recording the finest granularity of information. It is usually not enabled in production environments.
- Debug level records debugging information, generally used for performance analysis and problem positioning, usually not enabled in production environments.
- Info level records key information, suitable for monitoring the operational status of the system in production environments, without recording too many details.
- Warn level records potential issues, suitable for monitoring events that may need attention in production environments.
- Error level records error information, indicating that the program encountered an unresolvable error during execution, most commonly used as the log level in production environments. When error logs appear, they should be promptly addressed.

**Example**

```yml
logging:                                              # Log level
  level: info                                         # Run log level
```

## Alarm Log Configuration

The alarm log is the `metadata-validate-report-{unixts}.log` file located in `{DMC installation directory}/logs/{DMC instance name}/{migration task name}{timestamp}/warn`, where `{timestamp}` is the SCN formatted as `yyyyMMddHHmmss` and `unixts` is the standard SCN for Unix systems.

During the metadata synchronization phase, when metadata validation fails, this alarm log will be generated to alert users about which tables do not meet the requirements for data synchronization, for example:

```shell
Source table(YDS.TEST) column(ID, type: INTEGER) is not compatible with sink table(YDS.TEST) column(ID, type: VARCHAR(255 byte))
```

This log is generated by default and cannot be configured or modified.

## Metric Configuration

DMC provides the following metric data for users to monitor the state and progress of various events during the migration task:

**Metadata Synchronization Metrics**

Metadata synchronization metrics are mainly used to monitor the status and progress of tasks during the metadata phase.

**Full Data Synchronization Metrics**

Full data synchronization metrics are used to monitor the status and progress of tasks during the full data phase.

**Incremental Data Synchronization Metrics**

Incremental data synchronization metrics are used to monitor the status and real-time performance of tasks during the incremental data synchronization phase.

The above metric data is generated by default and cannot be configured or modified. Users can use the [show metrics](../DMC Introduction/User Interfaces.html#showmetrics) command in the client to view the metrics.

## Prometheus Integration Configuration

For guidance on integrating with Prometheus, please refer to the [Prometheus Integration Guide](https://cod-doc.yasdb.com/ecosystem/dev-yds/zh/Others/Prometheus.html) under the YashanDB ecosystem integration series. Additionally, the following configuration items should be added to the DMC server startup configuration file (`{DMC installation directory}/conf/yds_config.yml`):

```yml
plugins:
  prometheusExporter:
    port: "8101"
    enable: "true"
```
