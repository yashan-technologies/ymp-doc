**Specifications when RelayLog acts as the source end**

When RelayLog acts as the source, it uses the log data files generated during synchronization (when RelayLog served as the target end) as the data source.

Supported incremental event types include:

- DML Events
- Chunk Events

Data synchronization specifications:

- Only supports incremental mode.
- Only supports single-threaded concurrent reading.

Other specifications include:

- To ensure the integrity of data files under the basePath path, it is essential to maintain the integrity of files generated by RelayLog when acting as the target end, ensuring that the path contains the metadata file, checkpoint file, index file, and log file.
- The task name used as the source (RelayLog) must be consistent with the task name used as the target (RelayLog).

**Specifications when RelayLog acts as the target end**

When RelayLog acts as the target end, it serializes the events received from the source database into the corresponding data format and persists them to data files.

Supported incremental event types include:

- DML Events
- Chunk Events

Data synchronization specifications:

- Only supports incremental mode.
- Only supports single-threaded concurrent reading.
