## 服务器准备

### 服务器配置

|  环境配置项| 最低要求| 说明|
|:------ |:--------------------------|:------------------------------------------------------------------- |
| 操作系统   | CentOS 7.6以上、KylinOS V10、RHEL 9.3 |                                                                     |
| 处理器架构  | X86-64、ARM-64             | 处理器架构为ARM-64，YMP所需的JDK版本须为JDK17，能够有效解决可能出现的数据库连接慢问题。             |
| CPU核数  | 4核及以上                     |                                                                     |
| 可用内存   | 8G及以上                     | 根据数据库模型复杂度，对内存的使用情况不同，配置参考：[迁移常见问题22](../../常见问题/迁移任务.html#problem22) |
| 磁盘可用空间 | SSD，可用大小根据迁移表大小而定         | 建议不小于待迁移表中的最大单表数据量的3倍                                               |

> **Note**:
> 
> 部署环境配置至少满足最低要求。若配置未达最低要求，虽可以修改配置参数运行YMP系统，但在任务使用过程中可能出现卡顿，内存溢出等问题。

### 操作系统参数调整

为满足YMP默认内置库YashanDB数据库的正常安装和运行需要，建议按本文所列事项以root用户或具备sudo权限的用户登录服务器上进行配置调整。

将部分资源限制值（使用ulimit -a可查看所有的资源限制值）调整为推荐值或以上，详细信息如下表所示，其中ymp为规划的YMP安装用户示例（具体用户名需以实际规划为准）。

|  资源项| 推荐值| OS命令| 说明|
| :----------------- | :-------- | :----------------------------------------------------------- | :----------------- |
| open files         | 1048576   | ulimit -n 1048576 <br/> vi /etc/security/limits.conf后添加：<br/>ymp soft nofile 1048576<br/> ymp hard nofile 1048576 | 进程打开的文件数   |
| max user processes | 1048576   | ulimit -u 1048576<br/> vi /etc/security/limits.conf后添加：<br/> ymp soft nproc 1048576<br/>  ymp hard nproc 1048576 | 最大用户进程数     |
| max memory size    | unlimited | ulimit -m unlimited<br/>   vi /etc/security/limits.conf后添加：<br/> ymp soft rss unlimited<br/> ymp hard rss unlimited | 进程最大占用内存值 |
| stack size         | 8192      | ulimit -s 8192<br/> vi /etc/security/limits.conf后添加：<br/> ymp soft stack 8192<br/> ymp hard stack 8192 | 进程最大栈空间     |

### 开放端口

**安装默认端口号**

端口列表请参考：[端口列表](../../参考说明/端口列表)。

|  YMP监听| 数据库监听| 主机间通信| yasom| yasagent|
| ----- | ----- | ----- | ----- | -------- |
| 8090  | 8091  | 8092  | 8093  | 8094     |

**方式一：关闭防火墙**

在服务器上执行如下命令关闭防火墙：

```shell
# 关闭防火墙
systemctl stop firewalld 

# 关闭开机自启
systemctl disable firewalld
```

**方式二：添加白名单**

如果防火墙不能关闭，则需按如下步骤添加端口到白名单中：

1.查看防火墙端口开放情况。

```shell
firewall-cmd --zone=public --list-ports
```

2.添加端口到防火墙（Firewall）。

这里以8090为例演示如何添加端口到防火墙中，其他端口操作方法相同。

```shell
# 添加 --permanent 永久生效，没有此参数重启后失效
firewall-cmd --zone=public --add-port=8090/tcp --permanent

# 重新载入
firewall-cmd --reload

# 查看
firewall-cmd --zone=public --query-port=8090/tcp
```

如需从白名单中删除已添加的端口，使用如下命令：

```shell
# 删除已添加的端口
firewall-cmd --zone=public --remove-port=8090/tcp --permanent
```

## 用户准备

<span id="用户准备" class="yaslink"></span>

建议在服务器新建用户以安装部署YMP，注意YMP安装路径不可包含英文句号。

创建一个用户用于安装YMP，以ymp为例，在用户创建和授权后，后续所有安装步骤均在该用户下操作。

```shell
# 新建YMP用户
useradd -d /home/ymp -m ymp
passwd ymp
```

## 应用环境准备

### 服务器工具准备

```shell
# 安装lsof命令工具
yum install -y lsof
```

### JDK环境准备

YMP的JDK版本要求见：[支持的JDK](#supportedDependenceVersion)，支持通过Java官方路径下载上述版本的JDK并安装成功后，还需配置如下环境变量：

```shell
# 以ymp用户下JDK17安装路径为/usr/tools/jdk17为例
# su - ymp 
$ vi ~/.bash_profile

# 在文件结尾添加如下内容
export JAVA_HOME=/usr/tools/jdk17
export PATH=$JAVA_HOME/bin:$PATH
export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar

# 重新载入配置文件
$ source ~/.bash_profile

# 安装成功后查看JDK版本信息
$ java -version
```

### libaio环境准备

YMP运行需要libaio动态库。

```shell
# 查看是否已安装libaio动态库
rpm -qa | grep libaio

# 若未有版本信息打印，安装libaio
yum install -y libaio
```

### YashanDB环境准备（可选）

使用默认内置库时，本步骤可省略。

使用自定义内置库时：

* 如需一个全新的YashanDB单机环境，参考YashanDB官网文档进行安装部署。

* 如为一个已有的YashanDB单机环境，则需由DBA在该环境中执行如下脚本：

```sql
 -- 创建一个ymp用户（以YMP_DEFAULT为例）并为其授权
 CREATE USER YMP_DEFAULT IDENTIFIED BY Ymppw602. DEFAULT TABLESPACE users;
 GRANT ALL PRIVILEGES TO YMP_DEFAULT;
 GRANT DBA TO YMP_DEFAULT;
```

### DB2 client环境准备（可选）

如需要使用DB2到YashanDB的评估功能，请进行DB2 client环境安装：

1. 使用DB2 client包自行安装。

2. 使用ymp.sh中的[deploy指令](../../参考说明/指令说明.html#db2_deploy)启动可视化部署页面，使用该指令前需确保部署环境支持可视化操作。

3. 使用DB2作为源端时，需将YMP部署在DB2实例所在用户。

4. 安装成功后，修改conf/application.properties文件中的db2.installation.directory参数为DB2 client的安装路径。

5. DB2 client版本须与数据库版本严格保持一致。

请根据实际需求向我们的技术支持获取软件包：（xx.xx为软件版本号）

|  软件包名称| 说明|
| --------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| yashan-migrate-platform-xx.xx-linux-aarch64.zip <br/>yashan-migrate-platform-xx.xx-linux-x86-64.zip | YMP安装包，用于在Linux x86_64/arm64平台进行YMP的安装和部署。                         |
| yashandb-xx.xx-linux-x86_64.tar.gz<br/>yashandb-xx.xx-linux-aarch64.tar.gz                          | YashanDB安装包，用于在Linux x86_64/arm64平台下进行默认内置库的安装和部署，要求使用和迁移目标库相同的版本。 |
| instantclient-basic-linux.arm64-19.10.0.0.0dbru-2.zip<br/>instantclient-basic-linux.x64-19.19.0.0.0dbru.el9.zip | 默认情况下，YMP迁移工具已集成Oracle、MySQL、DM、YashanDB四个数据库源对应版本的JDBC驱动，迁移时默认通过JDBC接口迁移。如果Oracle到YashanDB数据迁移，GBK到GBK字符集场景下，可能需要整条迁移链路不做字符集转换迁移，可以使用replace指令添加/替换OCI依赖。OCI版本要求见：[支持的OCI](#supportedDependenceVersion)。 |

从本步骤开始的后续所有服务端安装步骤，将由YMP安装用户来进行操作，请先切换至YMP用户，或以YMP用户登录至服务器。

推荐安装路径为`/home/ymp`，请确保安装YMP时此目录为空。

将YashanDB安装包上传至ymp用户/home/ymp/路径下：

```shell
#使用root用户进行操作：修改安装包所属用户及用户组为ymp用户
chown ymp:ymp yashandb-xx.xx-linux-x86_64.tar.gz

# 从root用户切换至ymp用户
su - ymp

# 切换至安装路径
cd /home/ymp
```
