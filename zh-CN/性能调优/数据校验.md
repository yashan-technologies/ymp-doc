### 校验源端、目标端数据库性能优化

YMP的全量校验会从源端、目标端流式获取数据来进行对比，性能依赖于评估库并行执行性能，数据库的配置优化参考各个数据库性能优化手册。

### YMP数据校验调优配置参数

|  参数名| 默认值| 参数说明|
|-----------------------------------------|----------|-----------------------------|
| checkTask.task.parallel.max-num         | 20       | YMP平台最大并行执行的校验任务数。          |
| checkTask.datasource.max-connection     | 500      | YMP平台所有校验任务限制每个数据源支持的最大连接数。 |
| checkTask.datasource.connection-timeout | 10000    | YMP平台所有校验任务获取连接超时时间，单位：ms。  |
| 性能配置                                    | 50       | 设置最多有多少张表可以同时进行数据校验。        |
| 大表拆分并发度                                 | 10       |                             |
| 大表拆分行数                                  | 50000000 |                             |
| LOB类型一次获取长度                             | 1080     |                             |
| 不一致LOB类型差异展示长度                          | 200      |                             |

调优参考说明如下：

checkTask.task.parallel.max-num：

YMP平台最大允许同时执行校验任务的数目，通常情况下不会出现超过默认值的情况，若同时进行校验任务过多，可适当增加该配置。

checkTask.datasource.max-connection：

校验任务统一使用的连接池最多可获取的连接数，该连接池中连接为所有校验任务提供。若连接数不满足同时校验的表数，则任务无法启动。该参数和数据库本身配置相关，请确保该数目小于数据源最大可建立连接数。

checkTask.datasource.connection-timeout：

校验所用连接池获取连接的超时时间，若大于该时间无法获取连接则任务会出现错误。请根据数据库性能进行配置。

性能配置：

校验单个任务同时可以进行的表校验数目，若机器性能与数据库处理性能较优可增加并发度提升性能。请确保数据库可以提供足够连接数与足够的排序能力，ymp运行环境下有足够内存。

大表拆分并发度：

若校验一个表数目大于大表拆分行数，将对该表以大表拆分行数进行拆分比较，该并发度为最大允许单表拆分后可同时执行的单表子任务数目，注意事项与性能配置类型。

大表拆分行数：

判断一个表是否属于大表的范畴，若超过该数目则会对表以行数拆分为子任务进行校验。该数值与机器性能与数据库性能强相关，注意太小会产生过多消耗，产生过多数据库连接。

LOB类型一次获取长度：

大对象类型一次性的获取大小，blob类型为byte数组大小，clob类型为char数组大小。在机器给予YMP内存充足的情况下可以适当增加，提升流的拿取速度。

不一致LOB类型差异展示长度：

大对象类型的差异展示长度，由产生差异的偏移量与偏移量后的内容组成。该大小会消耗部分机器内存与性能，在确保可以识别差异时，越小越有利于性能提升。
